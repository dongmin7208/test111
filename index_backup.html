1. product page

상품 데이터를 웹페이지에 렌더링하는 함수인 renderProduct()를 사용하여 코드의 재사용성과 구조화를 높였습니다.

Lodash 라이브러리를 활용해 가격을 기준으로 오름차순 및 내림차순 정렬할 때도 renderProduct()를 호출하여
정렬된 데이터를 화면에 출력함으로써, 중복되는 DOM 생성 코드를 피하고 코드의 간결함을 유지했습니다.

이 구조는 이후 필터 기능이나 검색 기능을 추가할 때도 productList만 가공한 뒤 renderProduct()를 다시 호출하면 되도록

설계되어 있어, 유지보수와 기능 확장에 유리합니다.

2. join page

이 코드는 회원가입 페이지에서 사용자 입력값을 실시간으로 검증하고,
입력의 편의성을 높이며, 최종적으로는 올바른 데이터만 서버로 전달되도록 하는 코드 입니다.

1. 유효성 검사
- ID 길이 제한: 5~10자 사이여야 입력 가능
- 비밀번호 길이 제한: 8~15자
- 비밀번호 일치 확인: 비밀번호 확인 버튼 클릭 시 일치 여부 체크
- 이메일 형식 검사: @ 포함 + 도메인(.com, co.kr, .net) 포함 여부
- 빈칸일 경우 제출 제한: 마지막 완료 버튼 클릭 시 전체 항목을 검사

2. UX개선
- 휴대폰 번호 입력 시: 자동으로 다음 칸으로 이동

3. 마케팅 동의 체크박스
- 전체 동의 체크 시 개별 항목 3개도 체크됨
- 개별 체크 해제 시 전체 체크도 해제

4. 다음 주소 API 연동
- 주소 찾기 버튼 클릭 시 다음 우편번호 서비스 열기
- 선택한 주소를 자동으로 input 필드에 채워줌

5. 최종 제출
- 모든 입력값이 유효하고, 비밀번호 일치 확인까지 완료되었을 경우에만 회원가입 완료 처리
- 실패 시 입력 누락 알림 또는 비밀번호 확인 요청

3. login page

이 코드는 로그인 화면에서 사용자 인증 기능과 로그인 실패 제한, 비밀번호 가시성 토글,
그리고 아이디 및 비밀번호 찾기 모달 기능까지 포함한 종합 로그인 처리 코드 입니다.

1. 로그인 인증
- 입력한 아이디와 비밀번호가 미리 정의된 값과 일치하는지 확인

- 로그인 성공 시 메인 페이지로 이동

2. 에러 메시지 표시
- 로그인 실패 시 에러 메시지를 보여주고, 입력 필드에 다시 포커스하면 에러메시지를 숨겨 재입력을 돕는 UX처리

3. 비밀번호 표시/숨김 토글 기능

4. 모달 창을 활용한 아이디/비밀번호 찾기 기능
- 아이디 찾기, 비밀번호 찾기 버튼 클릭 시 각 모달창이 나타남
- 모달 내부에서 이름과 이메일, 또는 아이디와 이메일 입력값을 미리 정의된 값과 비교해 맞으면 아이디 또는 비밀번호를 알림으로 보여줌, 틀릴 경우 각 입력 필드 아래에 에러 메시지를 표시

사용 스택

#React ㅣ #Redux Toolkit ㅣ #styled-components ㅣ #React Bootstrap

사용 라이브러리

#Slick Slider

핵심 코드

1. redux

Redux Toolkit의 createSlice를 사용하여 장바구니(cart) 상태 관리 기능을 구현했습니다.

상품 추가, 삭제, 수량 증가/감소 등의 기능을 reducer로 구성했으며, findIndex를 활용해 해당 상품의 존재 여부를 확인한 후 상태를 효율적으로 업데이트하도록 구현했습니다.

2. styled-components

이 프로젝트에서는 스타일드 컴포넌트를 사용해 컴포넌트 단위로 스타일을 분리하고 재사용성을 높였습니다.

현재 보여지는 코드에서 Wrap, Gnb, Flex 등의 요소는 Flexbox와 여백을 활용해 레이아웃 정렬을 구현했고, StyledLink, HealLink는 React Router와 연동하여 활성화 상태에 따라 스타일이 변경되도록 처리했습니다.

이미지 및 텍스트 스타일도 통일감 있게 설정하여 UI의 일관성을 유지했습니다.

3. components

본 프로젝트는 React Router를 활용하여 구현하였으며, App.js 내에서 전체 라우팅을 구성하였습니다.

상단에는 Header, 하단에는 Footer 컴포넌트를 고정 배치하여 전역 레이아웃을 구성하였으며, 중앙에는 Routes를 통해
각 페이지를 동적으로 렌더링하도록 설계했습니다.

루트 경로 /에는 Home 컴포넌트를 렌더링하며, 해당 컴포넌트 내에 /cult와 /cart 하위 경로를 중첩 라우트 방식으로 구성하여 Culture와 Cart 컴포넌트를 각각 렌더링합니다.

또한, 상세 콘텐츠 페이지는 동적 라우팅을 적용하여 /:id 형식의 파라미터를 통해 각각의 콘텐츠 ID에 따라 CultureDetail, HealingDetail, EventDetail 컴포넌트가 동작하도록 구현하였습니다.

이를 통해 컴포넌트 기반의 재사용성과 유지보수성을 고려한 구조로 개발하였습니다.
