이러한 가치관이 끊임없이 변화하는 IT 업계와 닮았다고 생각하여, 이 분야에서 저의 역량을 마음껏 펼치고자 개발자의 길을 선택하게 되었습니다.

아이템 선정 사유

이러한 이유로 오프라인 멀티샵의 인기가 높아지고 있습니다.
온라인 또한 마찬가지라 생각합니다.
여러 브랜드의 정보를 한곳에서 제공함으로써 사용자의 니즈를 충족시킬 수 있다고 생각합니다.

경쟁사 분석

경쟁사 홈페이지를 분석 해보면 대부분 비슷한 구조를 가지고 있었습니다.
어느한 곳 특별함 없이 비슷한 구조를 가지고 있다는 것은 사용자가 느끼기에 이 구조가 쇼핑을 하기에 제일 적절하다는 내용을 알려줍니다.

사용자 요구사항 분석

여러 브랜드 중 트렌드가 되는 품목을 알고싶어 합니다.
상품의 사용 용도에 맞게, 그리고 여러 브랜드의 정보를 한번에 얻고싶어 합니다.
다른 사용자의 관심품목을 알고싶어 합니다.

핵심 코드

1. product page

상품 데이터를 웹페이지에 렌더링하는 함수인 renderProduct()를 사용하여 코드의 재사용성과 구조화를 높였습니다.

Lodash 라이브러리를 활용해 가격을 기준으로 오름차순 및 내림차순 정렬할 때도 renderProduct()를 호출하여 정렬된 데이터를 화면에 출력함으로써, 중복되는 DOM 생성 코드를 피하고 코드의 간결함을 유지했습니다.

이 구조는 이후 필터 기능이나 검색 기능을 추가할 때도 productList만 가공한 뒤 renderProduct()를 다시 호출하면 되도록 설계되어 있어, 유지보수와 기능 확장에 유리합니다.

2. join page

이 코드는 회원가입 페이지에서 사용자 입력값을 실시간으로 검증하고,
입력의 편의성을 높이며, 최종적으로는 올바른 데이터만 서버로 전달되도록 하는 코드입니다.

유효성 검사를 통해 사용자 경험을 향상시켰습니다.

2. responsive code

@media all and (min-width: 769px) 구간에서는 모바일 메뉴를 display: none으로 숨기고, PC 전용 메뉴를 display: block으로 활성화하여 화면 크기에 따라 서로 다른 메뉴 구성을 적용하였습니다.

축제야 놀자 React 프로젝트 기획서

프로젝트 개요

전국 축제 정보를 제공하는 React 기반의 SPA(Single Page Application)로, 사용자가 지역별, 시기별로 축제 정보를 쉽게 찾을 수 있도록 구성했습니다.

주요 기능

• 지역별 축제 검색
• 날짜별 축제 필터링
• 즐겨찾기 기능
• 반응형 디자인
• 상태 관리 (Redux Toolkit)

기술적 도전

React Router를 활용한 SPA 구현과 Redux Toolkit을 통한 전역 상태 관리, styled-components를 이용한 컴포넌트 스타일링에 중점을 두었습니다.

축제야 놀자 React 프로젝트 코드리뷰

사용 스택

#React | #Redux Toolkit | #styled-components | #React Bootstrap

사용 라이브러리

#Slick Slider

핵심 코드

1. redux

Redux Toolkit의 createSlice를 사용하여 장바구니(cart) 상태 관리 기능을 구현했습니다.

상품 추가, 삭제, 수량 증가/감소 등의 기능을 reducer로 구성했으며, findIndex를 활용해 해당 상품의 존재 여부를 확인한 후 상태를 효율적으로 업데이트하도록 구현했습니다.

2. styled-components

이 프로젝트에서는 스타일드 컴포넌트를 사용해 컴포넌트 단위로 스타일을 분리하고 재사용성을 높였습니다.

현재 보여지는 코드에서 Wrap, Gnb, Flex 등의 요소는 Flexbox와 여백을 활용해 레이아웃 정렬을 구현했고, StyledLink, HealLink는 React Router와 연동하여 활성화 상태에 따라 스타일이 변경되도록 처리했습니다.

이미지 및 텍스트 스타일도 통일감 있게 설정하여 UI의 일관성을 유지했습니다.
